# 多选释义模式 — 设计说明书

**日期**: 2026-05-28
**状态**: 已确认

## 概述

在答题页新增多选释义模式。用户需要在 4 个中文选项中选出该英文单词的**全部正确释义**，全部选对且无多选错误项才算正确。

## 放置位置

答题页顶部新增模式切换栏，单选题和多选题切换。默认单选题。

## 核心逻辑

### 选题范围
- 仅从有多条中文释义的单词中抽取（当前 102 个）
- 拆分中文释义：按 `，` `；` `、` 拆分为独立释义条目

### 选项生成
- 正确选项：该单词的所有中文释义（拆分后）
- 干扰选项：从其他单词随机抽取中文释义补齐到 4 项
- 选项随机排列

### 判定规则
- 全部正确释义选中 **且** 无错误项选中 → 正确
- 遗漏正确释义 **或** 选中了错误项 → 错误，进入错题本

### 交互流程
1. 显示英文单词 + 提示"请选出所有正确的中文释义"
2. 用户勾选选项（复选框样式，可多选）
3. 点击"确认提交"
4. 显示结果：绿色=选中且正确、黄色=遗漏的正确项、红色=选错的干扰项
5. 显示结果文字 + "下一题"按钮

## 视觉设计

- 选项使用方形复选框（区别于单选模式的圆形字母）
- 选中态：蓝色边框+蓝色背景
- 提交后：三色反馈（绿/黄/红）
- 底部图例说明三种颜色的含义

## 与现有功能联动

| 功能 | 多选模式行为 |
|------|-------------|
| 错题本 | 未完全正确 → 自动进入错题本 |
| 收藏本 | 答题时随时可收藏/取消收藏 |
| 复习模式 | 错题复习支持多选模式 |

## 实现要点

- 修改 `js/quiz.js`：新增多选渲染和判定逻辑
- 修改 `css/style.css`：新增复选框、提交按钮、三色反馈样式
- 复用已有数据：从现有 `App.words` 读取，拆分多义词的中文释义
- 模式状态存储在 `App` 中，切换 Tab 不丢失
